# ⚡ クイックスタートガイド

**10分でプロジェクトを始める** ためのガイドです。

---

## 🤖 このテンプレートの使い方

このテンプレートは **AIツールとの対話を通じてドキュメントを作成・改善していく** ことを前提に設計されています。

### 対応AIツール

| ツール | 用途 |
|--------|------|
| **Cursor** | コードエディタ統合、ドキュメント編集 |
| **Claude Code** | ドキュメント作成、分析、コード生成 |
| **GitHub Copilot** | コード補完、ドキュメント作成支援 |
| **OpenAI Codex** | コード生成、ドキュメント作成 |
| **Anthropic Claude** | 壁打ち、アイデア整理、分析 |
| **ChatGPT** | アイデア出し、壁打ち |

### 基本的なワークフロー

```
① あなたの頭の中にある曖昧なアイデア・仮説
    ↓
② AIに伝えて対話しながら整理
    ↓
③ ドキュメントとして構造化
    ↓
④ メモ・議事録をAIで整理してドキュメントを改善
    ↓
⑤ 繰り返し
```

### 具体例

**例1: 仮説を整理する**
```
あなた: 「30代のエンジニアで副業したい人向けのサービスを考えている。
        AIがプロジェクトをレコメンドしてくれるイメージ」

AI: 「整理すると以下のような仮説になりますね...
     - ターゲット: 30代エンジニア、副業に興味あり
     - 課題: 自分に合うプロジェクトを見つけられない
     - 解決策: AIによるプロジェクトレコメンド
     hypothesis.md に記載する形式にしましょうか？」
```

**例2: 議事録からドキュメントを改善する**
```
あなた: 「notes/meetings/2025-01-15-user-interview.md を読んで、
        仮説の検証結果を hypothesis.md に反映して」

AI: 「議事録を確認しました。以下の点を更新します：
     - H1（ターゲット仮説）: 検証済みに更新
     - H2（課題仮説）: 新しい課題を追加
     ...」
```

> 📖 詳細は [AI活用ガイド](docs/00_policy/ai-usage-guide.md) を参照

## Step 2: 自分自身を知る（5分）⭐ 推奨

> 💡 **なぜ最初にやるべきか**: 創業者の強み・経験・リソースを把握することで、AIがより的確なアドバイスを提供でき、現実的な事業計画を立てられます。

📄 `docs/01_discovery/founder-profile.md`

最低限これだけ埋める：
- **経歴・スキル**: あなたは何ができる人か
- **強み・弱み**: 活かせること、補う必要があること
- **原体験・動機**: なぜこの事業をやりたいのか
- **リソース**: 使える時間・資金・ネットワーク

```markdown
【一言で表すと】
例: B2B SaaS経験豊富なPMで、技術も分かるが営業は苦手。週末起業から始める。

【最大の強み】
例: プロダクト設計と顧客ヒアリングの経験

【最大の課題】
例: 営業・マーケティングの経験が浅い

【この事業をやる必然性】
例: 自分自身がこの課題で苦労した経験があり、解決策を知っている
```

## Step 3: 最初に埋める3つのファイル（5分）

### 1️⃣ 仮説を書く

📄 `docs/01_discovery/hypothesis.md`

```markdown
### H1: ターゲット仮説
| **仮説** | 〇〇な人がターゲットである |
| **根拠** | △△だから |
| **状態** | 🔴 未検証 |

### H2: 課題仮説
| **仮説** | ターゲットは「□□」という課題を持っている |
| **根拠** | △△だから |
| **状態** | 🔴 未検証 |
```

### 2️⃣ ターゲットを定義する

📄 `docs/01_discovery/target-persona.md`

最低限これだけ埋める：
- **誰？**: 年齢、職業、状況
- **何に困っている？**: 課題・ペイン
- **なぜ既存の解決策ではダメ？**

### 3️⃣ このREADMEを書き換える

📄 `README.md`

```markdown
# プロジェクト名

## 何を作るか
一言で説明

## 誰のために
ターゲット顧客

## 解決する課題
どんな課題を解決するか
```

## 次のステップ

### Discovery フェーズ

1. ターゲット顧客5人以上と話す
2. 仮説を検証して `hypothesis.md` を更新
3. 課題の優先順位をつける

### Strategy フェーズ

1. `mvp-definition.md` でMVPを定義
2. `roadmap.md` でスケジュールを決める
3. `business-model.md` で収益モデルを設計

### Development フェーズ

1. `requirements.md` で要件を整理
2. `tech-stack.md` で技術を選定
3. 開発開始 🚀
