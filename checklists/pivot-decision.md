## 3. ピボットのシグナル確認

以下のシグナルがあるかチェック

### ユーザー/市場のシグナル

- [ ] ユーザーが継続して使わない
- [ ] 有料転換率が極端に低い
- [ ] 解約率が高い
- [ ] ユーザーが「欲しい」と言うものと「実際に使う」ものが違う
- [ ] 競合が急激に強くなっている

### プロダクトのシグナル

- [ ] 作っている機能がユーザーに使われない
- [ ] 「これがあれば使う」と言われた機能を作っても使われない
- [ ] 根本的な技術的制約がある

### ビジネスのシグナル

- [ ] ユニットエコノミクスが成り立たない
- [ ] 市場規模が想定より小さい
- [ ] スケールが見込めない

### チームのシグナル

- [ ] チームのモチベーションが低下している
- [ ] ビジョンに共感できなくなっている

### ピボットの種類

どのタイプのピボットを検討しているか

- [ ] **ズームイン**: 1つの機能に特化
- [ ] **ズームアウト**: 機能を拡張してプラットフォーム化
- [ ] **顧客セグメント**: 別のターゲットに切り替え
- [ ] **顧客ニーズ**: 同じ顧客の別の課題を解決
- [ ] **プラットフォーム**: アプリからプラットフォームへ（またはその逆）
- [ ] **ビジネスモデル**: 収益モデルの変更
- [ ] **チャネル**: 販売・流通チャネルの変更
