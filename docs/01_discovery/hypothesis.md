# 仮説管理

## 概要

このドキュメントでは、プロジェクトの仮説を管理します。仮説は「ターゲット」「課題」「解決策」「価格」の4カテゴリで整理します。

> 【記入例】20-30代のフリーランスエンジニアで、副業で収入を増やしたいと考えている人がターゲットである

| 項目 | 内容 |
|------|------|
| **仮説** | （誰がターゲット顧客か） |
| **根拠** | （なぜそう考えるか） |
| **検証方法** | （どうやって確かめるか） |
| **成功基準** | （何をもって正しいとするか） |

## 参考: 仮説検証の進め方

### 1. 仮説を立てる
- 具体的で検証可能な形で記述
- 「〜だろう」ではなく「〜である」と断定形で

### 2. 検証方法を決める
- インタビュー（5-10人程度）
- アンケート（定量的な検証）
- プロトタイプテスト
- ランディングページテスト

### 3. 成功基準を決める
- 数値で判断できるようにする
- 例: 「10人中7人以上が課題を持っている」

### 4. 検証を実行
- 結果を客観的に記録
- 想定と違う結果も正直に記載

### 5. 結論を出す
- 検証済み → 次のフェーズへ
- 仮説が間違っていた → 仮説を修正して再検証
