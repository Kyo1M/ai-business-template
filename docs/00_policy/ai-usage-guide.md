# AI活用ガイド

## 概要

このプロジェクトではAIツール（Claude Code、Cursor、ChatGPT等）を活用してドキュメント作成・分析・開発を効率化します。

## 使用するAIツール

| ツール | 主な用途 |
|--------|----------|
| Claude Code | ドキュメント作成・編集、コードレビュー |
| Cursor | コード開発、ドキュメント編集 |
| ChatGPT | アイデア出し、壁打ち |
| Perplexity | リサーチ、競合調査 |

## AIとの連携方法

### 1. CLAUDE.md を活用

リポジトリルートの `CLAUDE.md` にはプロジェクトの概要とルールが記載されています。AIツールはこのファイルを参照してコンテキストを理解します。

### 2. プロンプト集を活用

`prompts/` フォルダに用途別のプロンプトがあります：

```
prompts/
├── ideation/     # アイデア発想
├── analysis/     # 分析
└── development/  # 開発
```

### 3. コンテキストを提供

AIに依頼する際は、関連ドキュメントを読み込ませると精度が上がります：

```
以下のドキュメントを踏まえて、XXXしてください：
- docs/01_discovery/hypothesis.md
- docs/01_discovery/target-persona.md
```

## フェーズ別AI活用

### Discovery（発見）フェーズ
- アイデアのブレインストーミング
- 仮説の言語化・構造化
- 競合リサーチの要約
- ペルソナの具体化

### Strategy（戦略）フェーズ
- バリュープロポジションの整理
- ビジネスモデルの検討
- MVP機能の優先順位付け
- ロードマップ作成

### Development（開発）フェーズ
- 要件定義書の作成
- 設計ドキュメント作成
- コード生成・レビュー
- テストケース作成

### Growth（成長）フェーズ
- データ分析・インサイト抽出
- 実験結果の解釈
- 改善案の提案
- レポート作成

## AIに依頼する際のコツ

### 良い依頼の例

```
【コンテキスト】
当プロジェクトはXXXをターゲットにした△△サービスです。
現在、以下の仮説を検証中です：（hypothesis.mdの内容）

【依頼】
上記を踏まえ、ユーザーインタビューで確認すべき質問を10個提案してください。

【出力形式】
- 質問文
- その質問で確認したい仮説
```

### 悪い依頼の例

```
インタビュー質問を考えて
```
（コンテキストがないため、汎用的すぎる回答になる）

## 注意事項

### やるべきこと
- AIの出力は必ず人間がレビューする
- 重要な意思決定はAI任せにしない
- 出典が必要な情報はファクトチェックする
- プロンプトの改善履歴を記録する

### やってはいけないこと
- 機密情報・個人情報をAIに入力しない
- AIの出力をそのまま確定情報として扱わない
- 著作権のある長文をそのまま入力しない

## プロンプトの管理

### 新しいプロンプトを追加する

1. `prompts/` の適切なフォルダに追加
2. 以下のフォーマットで作成：

```markdown
---
title: プロンプト名
purpose: 目的
input: 必要な入力
output: 期待する出力
---

# プロンプト名

## 使用方法
（いつ、どのように使うか）

## プロンプト本文
（実際のプロンプト）

## 使用例
（入出力の例）
```

3. `prompts/README.md` の一覧に追加

### プロンプトの改善

プロンプトを改善したら、ファイル内の「改善履歴」に記録：

```markdown
## 改善履歴

| 日付 | 変更内容 | 理由 |
|------|----------|------|
| 2025-01-15 | 出力形式を詳細化 | 曖昧な回答が多かったため |
```
